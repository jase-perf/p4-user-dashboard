# P4 User Dashboard

[English](README.md) | **日本語**

複数のPerforceサーバーにまたがるユーザー管理を行うための、軽量なローカル実行ツールです。ライセンス使用状況の監査、非アクティブアカウントの特定、大規模マルチサーバー環境でのユーザー管理を必要とする管理者向けに構築されています。

![ダッシュボード スクリーンショット](docs/screenshot-users.png)

## 課題

大規模なゲームスタジオや企業では、数十〜数百台のP4コミットサーバーを運用していることが珍しくありません。それらすべてのサーバーにまたがるユーザー管理は大きな負担です：

- 統合的なビューがない — 各サーバーが独立した情報の島になっている
- ライセンス監査のたびにP4Adminで各サーバーに個別接続が必要
- 同じ社員が複数のサーバーにアカウントを持ち、それぞれがライセンスを消費
- 真に非アクティブなユーザーの特定には、全サーバーのアクセス時刻を確認する必要がある

## このツールでできること

- **メールアドレスによるユーザー集約** — 設定した全P4サーバーのユーザーを一覧で表示
- **サーバー間の重複を可視化** — 複数サーバーにアカウントを持つユーザーと消費ライセンス数を確認
- **非アクティブアカウントの検出** — 「最終アクセス」と「全アカウント中の最古アクセス」列により、あるサーバーではアクティブだが別のサーバーでは放置されているユーザーを発見
- **全列でのフィルタリング・ソート** — 「〜より新しい」「〜より古い」の日付フィルターに対応
- **ユーザーの編集・削除** — 個別のユーザーアカウントの編集・削除、非SSOユーザーのパスワードリセット
- **混在環境に対応** — UnicodeサーバーとNon-Unicodeサーバーを自動判別して処理
- **日本語 / 英語の切り替え** — ページリロードなしでUI言語を切り替え可能

## クイックスタート

### 必要な環境

- Python 3.12+
- [uv](https://docs.astral.sh/uv/)（推奨）またはpip
- `p4` CLI（デモのサーバー管理コマンド用）
- 管理対象の各サーバーへのP4ログインチケット（`p4 login`）

### インストールと実行

```bash
git clone https://github.com/jase-perf/p4-user-dashboard.git
cd p4-user-dashboard

# 依存パッケージのインストール
uv sync            # または: pip install -r requirements.txt

# 設定ファイルの作成（下記「設定」セクションを参照）
cp demo/config.json my-config.json
# my-config.json を編集してサーバー情報を入力

# ダッシュボードの起動
uv run python dashboard.py my-config.json
# または: python dashboard.py my-config.json
```

ダッシュボードはローカルWebサーバーを起動し、ブラウザを自動で開きます。すべてのデータはローカルマシン上に保持されます。

### デモを試す

リポジトリには、5台の設定済みP4サーバーとリアルなテストデータを含むデモ環境が付属しています。`p4d` が必要ですが、セットアップスクリプトが自動でダウンロードできます：

```bash
# デモサーバーを起動（必要に応じてp4dをダウンロード）
bash demo/setup.sh

# デモサーバーに対してダッシュボードを起動
uv run python dashboard.py demo/config.json
```

デモには東京、大阪、名古屋、ソウル、シンガポールの5サーバーが含まれ、約60のユーザーアカウント（日本語、韓国語、中国語、インド、ベトナム、欧米の名前が混在）、1日前〜400日前までの様々なアクセス日時、UnicodeとNon-Unicodeサーバーの混在環境が用意されています。

```bash
# デモサーバーの状態を確認
bash demo/setup.sh --status

# デモサーバーを停止してクリーンアップ
bash demo/setup.sh --teardown
```

`p4d` を手動でダウンロードする場合は、[perforce.com/downloads](https://www.perforce.com/downloads/helix-core-server) から取得し、`demo/bin/` に配置してください。

## 設定

ダッシュボードはJSON設定ファイルを読み込みます：

```json
{
  "port": 8181,
  "licensedUniqueUsers": 1500,
  "servers": [
    {"name": "prod-tokyo", "port": "ssl:p4-tokyo:1666", "user": "admin"},
    {"name": "dev-osaka", "port": "ssl:p4-osaka:1666", "user": "p4admin"},
    {"name": "art-nagoya", "port": "ssl:p4-nagoya:1666"}
  ]
}
```

| フィールド | 説明 |
|---|---|
| `port` | Web UIのローカルポート（デフォルト: 8080） |
| `licensedUniqueUsers` | 任意 — トップバーのインジケーター用のライセンスユニークユーザー総数 |
| `servers[].name` | サーバーの表示名 |
| `servers[].port` | P4PORT接続文字列 |
| `servers[].user` | P4ユーザー名（任意 — 省略時はシステムデフォルト） |

設定はダッシュボードの「Config」タブからも編集できます。

### 認証

ダッシュボードは既存のP4ログインチケットを使用します — `p4` やP4Adminで使用するものと同じ認証情報です。接続前に：

```bash
p4 -p ssl:p4-tokyo:1666 login    # 各サーバーにログイン
p4 -p ssl:p4-osaka:1666 login
```

サーバーが認証を要求した場合、ダッシュボードにエラーが表示されます。ログイン後にリトライでき、再起動は不要です。

## アーキテクチャ

```
ブラウザ (localhost)             Pythonサーバー (FastAPI)          P4サーバー
┌──────────────────┐           ┌──────────────────────┐         ┌──────────┐
│ 全データセットを  │   HTTP    │ ユーザーデータを     │  P4API   │ サーバーA │
│ JSONで保持 —     │◄─────────►│ 取得・集約、メモリに │◄────────►│ サーバーB │
│ フィルタ/ソートは│           │ キャッシュ、書込み   │         │ サーバーC │
│ クライアント側で │           │ 操作をプロキシ       │         │ ...      │
└──────────────────┘           └──────────────────────┘         └──────────┘
```

- **ローカル実行** — LAN アクセス用に `0.0.0.0` にバインド、クラウド依存なし
- **データベース不要** — 起動時/リフレッシュ時にP4サーバーから取得し、メモリにキャッシュ
- **クライアントサイドのパフォーマンス** — 全データセットをブラウザに送信、数千ユーザーでも瞬時にフィルタリング・ソート
- **Unicode自動検出** — UnicodeとNon-UnicodeのP4サーバーを自動的に処理

## 開発

```bash
# 開発用依存パッケージのインストール
uv sync

# テスト用にデモサーバーを起動
bash demo/setup.sh

# テスト実行
uv run pytest tests/ -v

# デモサーバーの停止
bash demo/setup.sh --teardown
```
